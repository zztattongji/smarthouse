# CI1302 语音模块说明

本目录用于说明项目中 CI1302 语音芯片的作用、SDK 来源、协议对接方式和烧录注意事项。由于 CI1302 原厂 SDK、模型文件、语音资源和固件包可能涉及厂商授权，本仓库不直接上传完整 SDK，只保留二次开发所需的接口说明和联调记录。

## 模块定位

在本项目中，CI1302 不是外部智能家居节点，也不是直接负责 WiFi/SLE 通信的芯片。它主要承担语音交互入口的职责：

- 语音唤醒：识别唤醒词，进入语音交互状态。
- 离线识别：识别固件中预置的离线命令词或语义 ID。
- 语音播报：播放本地提示音或反馈音。
- 串口通信：将识别结果、状态变化或语义事件发送给 WS63。

WS63 则负责联网、云端/大模型能力、外部智能家居节点通信，以及将 CI1302 的语义结果转换为统一 JSON/语义控制协议。

## 使用的 SDK

当前项目参考的 CI1302 SDK 为：

```text
CI13XX_SDK_LLM_AIoT_V1.0.5_Tuya_V1.0
```

本地路径示例：

```text
D:\AI_language\启英泰伦+涂鸦离在线大模型对话方案开发套件包\T3-V-PRO-EVB_语音SDK包\CI13XX_SDK_LLM_AIoT_V1.0.5_Tuya_V1.0
```

该 SDK 属于 CI13XX 系列离在线语音 AIoT 方案，包含离线 ASR、语音资源、固件生成工具和涂鸦/AIoT 相关能力。由于项目目标是保留原厂离在线对话能力，并在 WS63 侧扩展外部智能家居控制，因此不建议直接替换或公开上传完整 CI1302 SDK。

## 与 WS63 的关系

项目中的语音中枢由 CI1302 + WS63 组成，二者需要通过板载连接进行协同。根据目前已查阅的规格书、博客和 CI13XX 协议资料，较可能的对接方式是 UART，但具体串口号、引脚、波特率和是否存在私有握手仍需结合原厂资料确认。

推荐的逻辑边界如下：

```text
用户语音
   |
   v
CI1302：唤醒 / 离线识别 / 播报
   |
   | 识别结果、语义 ID、状态通知
   v
WS63：协议解析 / 语义映射 / 联网能力 / 外部节点通信
   |
   | 统一 JSON/语义控制协议
   v
网关节点 / 检测节点 / 执行节点 / App / 华为云
```

因此，CI1302 侧更适合保持原厂语音能力稳定，WS63 侧作为二次开发扩展入口。

## 协议参考

当前已参考的 CI13XX 系列协议包括：

- CI13XX 系列芯片串口协议 V1
- CI13XX 系列芯片串口协议 V2
- CI13XX 系列芯片 IIC 协议说明

### 串口协议 V1 摘要

V1 协议通常为固定长度帧：

```text
Head    ID    Cmd    Data_L    Data_H    CheckSum    End
A5 FA   00    81     xx        xx        sum         FB
```

常见含义：

- `0xA5 0xFA`：帧头
- `ID`：产品 ID，默认 0
- `Cmd`：命令码，`0x80` 以上通常为语音芯片发送给主控
- `Data`：命令词 ID 或控制数据，低字节在前
- `CheckSum`：前面字段累加和
- `0xFB`：帧尾

### 串口协议 V2 摘要

V2 协议支持变长消息，包含帧头、长度、消息类型、命令号、序号、有效数据、校验和和帧尾。示例结构可概括为：

```text
A5 FC | Len_L Len_H | MsgType | Cmd | Seq | Payload | CheckSum_L CheckSum_H | FB
```

其中语音识别结果可能通过命令号上报语义 ID 或命令词信息。V2 默认波特率通常为 `9600`，默认串口通常为 `UART1`，但实际工程可能由 `user_config.h` 或厂商方案重新配置。

## 当前联调判断

项目调试中曾出现以下现象：

- WS63 固件可以正常编译和烧录。
- 烧录自定义 WS63 固件后，原有离在线语音对话能力可能失效。
- 设备仍能唤醒，但后续对话、播报或语义响应异常。

这说明原厂方案中的 WS63 很可能不只是“无线通信芯片”，还承担了与 CI1302 的协议握手、状态同步、联网大模型交互、音频上传下载或播放控制等逻辑。因此，如果希望保留原厂 AI 对话能力，不能只按普通 WS63 Demo 方式替换固件。

## 烧录说明

CI1302 侧常用烧录工具为：

```text
PACK_UPDATE_TOOL.exe
```

本地工具路径示例：

```text
D:\AI_language\启英泰伦+涂鸦离在线大模型对话方案开发套件包\T3-V-PRO-EVB_语音SDK包\CI13XX_SDK_LLM_AIoT_V1.0.5_Tuya_V1.0\tools\PACK_UPDATE_TOOL.exe
```

烧录前需要确认：

- 板卡进入 CI1302 下载模式，而不是 WS63 下载模式。
- 串口连接到 CI1302 下载串口。
- 电源、GND、TX、RX、RST/BOOT 等引脚连接正确。
- 选择的固件包与当前板卡、Flash 分区和语音资源匹配。

如果不确定固件是否匹配，不建议反复烧录未知固件。更稳妥的做法是先备份或索取原厂恢复包。

## 后续需要向原厂确认的问题

为保证 CI1302 + WS63 方案可稳定二次开发，建议继续向原厂确认：

1. CI1302 与 WS63 实际通信方式是否为 UART。
2. 对应 UART 号、引脚、波特率、数据位、停止位和校验方式。
3. CI1302 上报唤醒、离线识别、语义 ID、播放状态时使用标准 CI13XX 协议还是私有协议。
4. WS63 原厂工程中是否包含联网大模型、涂鸦平台、音频上传下载、心跳 ACK 和状态同步逻辑。
5. 是否可以提供语义结果回调、串口透传、自定义命令转发或外设控制扩展接口。
6. 是否可以提供 WS63 原厂固件恢复包和 CI1302/WS63 联调示例工程。

## 仓库保留方式

本仓库只保留 CI1302 的接口说明和联调结论，不上传：

- 原厂 CI13XX SDK 完整源码
- 离线识别模型
- 语音资源包
- 固件生成中间产物
- 可直接烧录的原厂固件
- 厂商协议中未公开的私有资料

这样既能保证项目资料完整，又能避免公开仓库中出现授权或敏感资料问题。
